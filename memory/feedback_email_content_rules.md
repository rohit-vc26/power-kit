# Email Content Rules — Power-Kit Standard

**For team members writing emails using Power-Kit.**

These rules are non-negotiable. Power-Kit checks them automatically.

---

## Rule 1: AI Detection-Free (Humanized)

Content must pass AI detection (Originality.ai, GPTZero).

**Check:**
- Uses contractions (we've, don't, you'll)
- Active voice (not passive)
- Specific details (names, dates, situations)
- Varied sentence length
- One unique phrase

---

## Rule 2: NO Em-Dashes

Em-dash `—` is banned. Spam filters flag them. Encoding errors break them.

**Use instead:** comma, parentheses, period, hyphen.

---

## Rule 3: 65-90 Words Per Section

- Below 65 = cheap, low-effort
- Above 90 = too long
- 65-90 = sweet spot

Measure with wordcounter.net.

---

## Rule 4: CTA + Below-CTA Hook

Every email needs both:
1. Clear button/link CTA
2. Hook below (curiosity/social proof/reassurance/FOMO/benefit)

---

## Rule 5: Follow Power-Kit Standards

- No secrets in copy
- No customer data exposed
- HTTPS links only
- Verify before send

---

## Rule 6: NO Spam Words

Banned list:
```
guaranteed, guarantee, warranties, warranted,
risk-free, no risk, limited time, act now,
click here immediately, don't miss out, claim your,
free money, one-time offer, exclusive, last chance,
must work, can't fail, 100% success, miracle,
amazing results, unbelievable
```

Replace with: "most see", "available this week", "get started today", "research shows".

---

## Rule 7: Real Promises Only

Test: Would you say this face-to-face to the customer?

If NO → delete that sentence.

---

## Rule 8: NO Guarantees

Banned: guarantee, warrant, ensures, must work, 100% success.

Replace with: "most users see", "customers have gotten", "your team can", "based on data from".

---

## Enforcement

Power-Kit auto-checks emails on Write/Edit. Violations show warnings. Team is expected to fix before send.
